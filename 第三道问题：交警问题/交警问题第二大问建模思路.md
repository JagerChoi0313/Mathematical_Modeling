第二大问的意思可以这样理解：**第 32 个节点 P 点发生重大刑事案件，嫌疑人开车跑了，而且警方晚了 3 分钟才接到报警。现在要用“全市所有交巡警服务平台”的警力，尽快把嫌疑人围住，不让他继续逃出控制范围。** 题目原文强调的是“地点 P（第 32 个节点）”“案发 3 分钟后接到报警”“调度全市交巡警服务平台警力资源”“最佳围堵方案”。

这道题不是简单问“最近的警察去哪抓人”，而是一个**全市范围内的动态围堵调度问题**。

------

## 一、第二问的核心场景

可以把时间线想清楚：

**案发时刻：0 分钟**

嫌疑人从第 32 个节点 P 出发，开车逃跑。

**报警时刻：3 分钟**

警方这时候才知道案件发生。

也就是说，警方开始行动时，嫌疑人已经提前跑了 3 分钟。

题目默认警车速度是第一问给出的 **60 km/h**，也就是：

**1 km/min**

如果嫌疑人也按相同速度逃跑，那么他在报警前已经沿路网跑出了大约 **3 km 的路程范围**。

所以第二问最关键的一句话是：

> 警方不是从案发瞬间开始追，而是落后了嫌疑人 3 分钟。

------

## 二、题目真正想让我们做什么？

它想让我们回答：

> 在嫌疑人已经逃跑 3 分钟的情况下，应该从哪些交巡警平台派警力，分别去哪些路口或节点设卡，才能最快形成一个包围圈？

也就是说，最后论文里要给出类似这样的结果：

| 调度平台 | 前往围堵节点/路口 | 到达时间 | 作用             |
| -------- | ----------------- | -------- | ---------------- |
| 平台 A   | 节点 x            | 2.4 分钟 | 封锁北侧逃跑路线 |
| 平台 B   | 节点 y            | 3.1 分钟 | 封锁东侧逃跑路线 |
| 平台 C   | 节点 z            | 2.8 分钟 | 封锁南侧逃跑路线 |

当然，实际节点编号要通过计算得到。

------

## 三、和第一大问的区别

第一大问主要研究的是 **A 区内部**：

1. 给 20 个平台分配管辖范围；
2. 对 A 区 13 条交通要道进行封锁；
3. 判断是否新增平台。

第二大问变成了 **全市六区范围**：

它不再局限于 A 区，也不是只封锁 13 条进出 A 区的道路，而是要看嫌疑人从 **第 32 个节点** 出发后，可能沿着全市交通网络往哪里逃。

所以第二问的范围更大，问题也更动态。

------

## 四、“最佳围堵方案”是什么意思？

这里的“最佳”一般可以理解为：

**在保证嫌疑人不会逃出包围圈的前提下，使围堵完成时间尽可能短。**

换句话说，不是派警察越多越好，而是要合理安排：

1. 哪些平台出警；
2. 去哪些关键节点封锁；
3. 每个平台最多封锁一个节点或路口；
4. 所有关键逃跑方向都要被堵住；
5. 最晚到达的那个封锁点时间要尽量小。

所以它本质上是一个：

**最短路径分析 + 围堵节点选择 + 警力分配优化问题。**

------

## 五、这道题应该怎么理解嫌疑人的逃跑范围？

从第 32 个节点出发，嫌疑人可能沿任意道路逃跑。

报警时已经过去 3 分钟，所以嫌疑人可能已经到达所有满足下面条件的节点：

**从 32 号节点到该节点的最短行驶时间 ≤ 3 分钟**

这些节点可以理解为：

> 报警时嫌疑人可能已经到达或经过的区域。

接下来警方要做的，不一定是在这个区域里抓他，而是要在这个区域外面选择一些关键路口设卡，形成封锁圈。

------

## 六、为什么要用“路网最短路径”？

因为嫌疑人和警车都不是直线飞过去的，而是沿交通网络行驶。

所以要先把整个城市交通网络看成一个图：

- 节点：交通路口；
- 边：道路；
- 边权重：道路长度或通行时间；
- 平台：分布在某些节点上的警力点；
- P 点：第 32 个节点。

然后分别计算：

1. **嫌疑人从 32 号节点到所有节点的最短时间；**
2. **每个交巡警平台到所有节点的最短时间。**

这一步通常用 **Floyd 算法** 或 **Dijkstra 算法**。

如果节点数量不是特别大，用 Floyd 比较方便；如果节点多，用 Dijkstra 更高效。

------

## 七、第二问最后要输出什么？

论文里第二问一般要输出三类东西。

第一类是：**嫌疑人可能逃跑范围**

比如：

> 报警时嫌疑人可能已经到达节点 32 周围若干节点，形成初始危险区域。

第二类是：**围堵点选择**

也就是在哪些节点设卡。

这些节点应该满足一个要求：

> 嫌疑人想从 P 点继续往外逃，必须经过这些封锁节点中的某些节点。

这在图论里可以理解为“割点”“割集”或“边界封锁点”。

第三类是：**平台调度方案**

也就是：

> 哪个平台去哪个节点，多久能到，是否能赶在嫌疑人之前完成封锁。

------

## 八、一个非常重要的判断条件

假设某个围堵节点是 (j)。

嫌疑人从 P 点到节点 (j) 的最短时间是：

[
T_s(j)
]

某个平台 (i) 到节点 (j) 的最短时间是：

[
T_p(i,j)
]

因为警方晚了 3 分钟才接警，所以警察到达节点 (j) 的绝对时间是：

[
3 + T_p(i,j)
]

要想在嫌疑人之前封锁这个节点，就要满足：

[
3 + T_p(i,j) \leq T_s(j)
]

也可以写成：

[
T_p(i,j) \leq T_s(j) - 3
]

这句话特别重要，论文里可以作为第二问的核心约束。

意思是：

> 警察虽然晚出发 3 分钟，但只要围堵点选得足够远，警察仍然可能提前到达并完成封锁。

------

## 九、用一句话概括第二问

第二大问就是：

> 以第 32 个节点为嫌疑人逃跑起点，考虑警方接警延迟 3 分钟，在全市交通网络中计算嫌疑人可能的逃跑路径，并调度全市交巡警平台，在关键节点形成最快、有效的围堵封锁方案。

## 十、你写论文时可以这样描述“问题二”

可以直接这样理解和写：

问题二研究的是重大刑事案件发生后犯罪嫌疑人的动态逃逸围堵问题。案件发生地点为该市第 32 个交通节点，且警方在案发 3 分钟后才接到报警，因此在警方开始调度警力时，嫌疑人已经沿交通网络行驶了一定距离。为尽快控制嫌疑人的逃逸范围，需要根据全市六区交通网络结构、各交巡警服务平台位置以及道路通行时间，分析嫌疑人可能到达的区域，并选择若干关键路口作为围堵点，调度相应平台警力前往封锁。该问题的目标是在保证主要逃逸路径被有效阻断的前提下，使围堵完成时间尽可能短，从而得到全市交巡警服务平台警力资源的最优调度方案。

后面正式建模时，第二问可以按这条线走：**先算嫌疑人逃逸范围，再找关键封锁节点，最后做人车平台匹配调度。**

可以。第二大问建议不要直接写成“追捕问题”，而要写成：

> **基于全市交通网络的重大刑事案件逃逸围堵优化模型。**

题目给出的关键信息是：案件发生在 **P 点，即第 32 个节点**；警方在 **案发 3 分钟后** 才接警；此时嫌疑人已经驾车逃跑；要求调度 **全市交巡警服务平台**，给出最佳围堵方案。

# 一、第二问总体建模思路

第二问可以分成三步：

**第一步：计算嫌疑人可能逃逸范围。**

嫌疑人从第 32 个节点出发，已经提前逃跑 3 分钟，所以要先算出：

> 从 32 号节点出发，3 分钟内嫌疑人可能到达哪些节点。

这些节点构成“初始逃逸区域”。

**第二步：确定围堵圈。**

嫌疑人可能继续往外逃，所以不能只盯着 3 分钟范围内的节点，而要在它外围找一圈关键节点。

这些关键节点应该满足：

> 嫌疑人从 P 点继续向外逃时，必须经过这些节点或道路。

这一圈节点就是候选围堵点。

**第三步：把交巡警平台分配到围堵点。**

对每个候选围堵点，计算各个平台到它的最短时间，然后选择合适的平台去封锁，使整体围堵完成时间最短。

所以整体逻辑是：

> **逃逸范围分析 → 围堵点筛选 → 平台调度匹配 → 输出最佳围堵方案。**

------

# 二、模型一：全市交通网络最短路径模型

先把全市交通网络抽象成图：

[
G=(V,E)
]

其中：

- (V)：全市所有交通节点；
- (E)：节点之间的道路；
- (d_{ij})：节点 (i) 到节点 (j) 的道路长度；
- (t_{ij})：节点 (i) 到节点 (j) 的通行时间。

题目中警车速度为：

[
60km/h=1km/min
]

所以如果道路长度单位是 km，则：

[
t_{ij}=d_{ij}
]

也就是说，**路长多少 km，通行时间就是多少分钟**。

然后用 **Floyd 算法** 或 **Dijkstra 算法** 求全市任意两个节点之间的最短通行时间。

这里建议继续沿用第一问的 Floyd 算法，因为第一问你已经用过，论文逻辑会更统一。

记：

[
D_{ij}
]

表示节点 (i) 到节点 (j) 的最短通行时间。

------

# 三、模型二：嫌疑人逃逸范围模型

嫌疑人从第 32 个节点出发。

设嫌疑人到达节点 (j) 的最短时间为：

[
T_s(j)=D_{32,j}
]

由于警方在案发 3 分钟后才接警，所以报警时嫌疑人可能已经到达的节点集合为：

[
R_0={j\mid T_s(j)\leq 3}
]

这个集合 (R_0) 就是：

> 嫌疑人在警方接警前可能已经到达的区域。

这一部分在论文里可以叫：

**初始逃逸区域判定模型。**

------

# 四、模型三：候选围堵点筛选模型

接下来要考虑：嫌疑人已经跑出去了，警方不能只在 (R_0) 里面找人，而是要在外面设卡。

所以可以构造一个“逃逸扩散圈”。

设时间阈值为 (\tau)，嫌疑人在 (\tau) 分钟内可能到达的节点集合为：

[
R(\tau)={j\mid T_s(j)\leq \tau}
]

其中：

[
\tau>3
]

当 (\tau=4,5,6,\dots) 时，嫌疑人可能到达的范围越来越大。

围堵点可以选在 (R(\tau)) 的边界上。

边界节点可以定义为：

[
C(\tau)={j\notin R(\tau)\mid \exists k\in R(\tau),(k,j)\in E}
]

意思是：

> (j) 不在嫌疑人当前可能到达范围内，但它和逃逸范围内的某个节点直接相连。

这些节点就是嫌疑人继续向外逃时可能经过的“外侧边界节点”，适合作为候选围堵点。

简单说：

- (R(\tau))：嫌疑人可能活动区域；
- (C(\tau))：嫌疑人继续外逃必须突破的边界；
- 调度平台去 (C(\tau)) 设卡，就能形成围堵圈。

------

# 五、模型四：平台到围堵点的调度匹配模型

设全市交巡警平台集合为：

[
S={1,2,\dots,m}
]

设平台 (i) 所在节点为 (p_i)。

平台 (i) 到候选围堵点 (j) 的最短时间为：

[
T_p(i,j)=D_{p_i,j}
]

但是警方是案发 3 分钟后才开始行动的，所以平台 (i) 到达围堵点 (j) 的绝对时间是：

[
3+T_p(i,j)
]

嫌疑人最早到达节点 (j) 的时间是：

[
T_s(j)
]

因此，要想成功封锁节点 (j)，必须满足：

[
3+T_p(i,j)\leq T_s(j)
]

这就是第二问最关键的约束。

它的含义是：

> 虽然警察晚出发 3 分钟，但只要警察能在嫌疑人到达该围堵点之前赶到，就能完成有效围堵。

------

# 六、0-1 规划模型

定义决策变量：

[
x_{ij}=
\begin{cases}
1, & \text{平台 } i \text{ 被派往围堵点 } j\
0, & \text{否则}
\end{cases}
]

每个围堵点必须有一个平台负责：

[
\sum_{i\in S}x_{ij}=1,\quad j\in C(\tau)
]

每个平台最多封锁一个节点：

[
\sum_{j\in C(\tau)}x_{ij}\leq 1,\quad i\in S
]

有效围堵约束为：

[
3+\sum_{i\in S}x_{ij}T_p(i,j)\leq T_s(j),\quad j\in C(\tau)
]

目标函数可以设为：

[
\min Z=\max_{j\in C(\tau)}\left(3+\sum_{i\in S}x_{ij}T_p(i,j)\right)
]

也就是：

> 最小化所有围堵点中最晚完成封锁的时间。

这个目标非常合理，因为围堵是否成功，取决于最慢到达的那个关键封锁点。

------

# 七、最佳围堵方案的判断标准

建议你把“最佳”定义成三层：

## 第一层：必须能堵住

所有关键围堵点都要满足：

[
3+T_p(i,j)\leq T_s(j)
]

否则嫌疑人可能比警察先到，这个点就堵不住。

## 第二层：围堵完成时间最短

在所有可行方案中，让最晚封锁完成时间最小。

也就是：

[
\min Z
]

## 第三层：安全裕度尽可能大

安全裕度可以定义为：

[
M_j=T_s(j)-[3+T_p(i,j)]
]

其中 (M_j) 越大，说明警察越早于嫌疑人到达，围堵越安全。

如果有多个方案的最晚完成时间差不多，可以选择最小安全裕度最大的方案：

[
\max \min M_j
]

------

# 八、具体求解流程

论文里可以按这个流程写。

## 第一步：数据预处理

根据附件 2 读取：

- 全市交通节点数据；
- 节点之间的道路连接关系；
- 各交巡警服务平台所在节点。

构建全市交通网络邻接矩阵。

如果两个节点之间有道路，则填入道路长度；如果没有道路，则记为无穷大。

------

## 第二步：求全市最短路径矩阵

用 Floyd 算法计算所有节点之间的最短时间矩阵：

[
D=(D_{ij})
]

其中 (D_{ij}) 表示节点 (i) 到节点 (j) 的最短通行时间。

------

## 第三步：计算嫌疑人逃逸时间

以第 32 个节点为起点，提取第 32 行最短路径数据：

[
T_s(j)=D_{32,j}
]

得到嫌疑人到达每个节点的最短时间。

------

## 第四步：确定报警时逃逸范围

计算：

[
R_0={j\mid T_s(j)\leq 3}
]

这个集合就是案发 3 分钟后嫌疑人可能已经到达的节点。

------

## 第五步：逐层寻找可行围堵圈

从 (\tau=4) 分钟开始，逐步扩大嫌疑人逃逸范围：

[
R(4),R(5),R(6),\dots
]

对每一个 (\tau)，找出边界节点集合：

[
C(\tau)
]

然后判断：

> 是否可以调度交巡警平台在嫌疑人到达这些边界节点前完成封锁。

如果不能，就继续向外扩大一圈。

如果能，则该 (C(\tau)) 就是一个可行围堵圈。

------

## 第六步：求平台—围堵点最优匹配

对候选围堵点 (C(\tau))，建立平台到围堵点的时间矩阵：

[
T_p(i,j)
]

然后用 0-1 规划或匈牙利算法分配平台，使：

[
\max \text{到达时间最小}
]

得到每个平台具体去哪个节点。

------

## 第七步：输出最佳围堵方案

最后输出：

| 围堵点 | 负责平台 | 平台所在节点 | 平台到达时间 | 嫌疑人最早到达时间 | 是否可提前封锁 |
| ------ | -------- | ------------ | ------------ | ------------------ | -------------- |
| 节点 j | 平台 i   | 节点 p_i     | (3+T_p(i,j)) | (T_s(j))           | 是             |

这一张表就是第二问最重要的结果表。

------

# 九、第二问推荐的模型名称

你可以在论文里把第二问模型命名为：

**基于最短路径与动态边界搜索的交巡警围堵调度模型**

或者更简洁一点：

**重大刑事案件动态围堵调度模型**

模型结构可以写成：

1. 全市交通网络最短路径模型；
2. 嫌疑人逃逸范围判定模型；
3. 边界围堵点筛选模型；
4. 平台—围堵点 0-1 调度优化模型。

# 十一、这道题最适合的建模主线

你可以按下面这个主线来写第二问：

> 首先，将全市交通网络抽象为带权无向图，并利用 Floyd 算法求出任意节点之间的最短通行时间。然后，以第 32 个节点为嫌疑人逃逸起点，计算嫌疑人在接警前 3 分钟内可能到达的初始逃逸区域。接着，按照嫌疑人逃逸时间逐层扩展逃逸范围，提取逃逸区域外侧边界节点作为候选围堵点。最后，以围堵完成时间最短为目标，建立平台—围堵点 0-1 分配模型，确定各交巡警服务平台的具体调度方案，从而实现对嫌疑人的快速围堵。

------

# 十二、简单说，第二问就用这个模型组合

最推荐的建模组合是：

[
\text{Floyd 最短路径}+\text{逃逸范围扩散}+\text{边界围堵点筛选}+\text{0-1 规划调度}
]

这个组合最稳，也最适合数学建模论文。
