好，我们现在**从零开始重新理解**，前面那些代码、公式、论文都先放一边。你把自己当成第一次接触数学建模的人，我们先搞清楚三件事：

第一，什么是数学建模；
 第二，数学建模一般怎么做；
 第三，这道露天矿车辆安排题到底怎么一步步建成数学模型。

------

# 一、什么是数学建模？

数学建模说白了就是：

> 把一个现实问题，翻译成数学问题，再用数学方法求解，最后把结果翻译回现实方案。

比如这道题现实中问的是：

> 露天矿里有电铲、有卡车、有铲位、有卸点，怎么安排才能完成生产任务，并且运输成本低或者产量高？

这句话本身不是数学。
 我们要把它变成数学语言：

```
哪些铲位放电铲？
每条路线派几辆车？
每条路线运输多少趟？
总运量是多少？
产量是否达标？
矿石品位是否合格？
```

然后用变量、目标函数、约束条件把它写出来。

所以数学建模的核心不是一上来写公式，而是：

> 先看懂现实问题，再决定用什么变量描述它，最后用公式限制它、优化它。

------

# 二、数学建模的一般步骤

数学建模一般可以按这 6 步来做。

## 第一步：读题，明确问题要你“决定什么”

这是最重要的一步。

这道题不是单纯算一个数，而是要我们做一个**安排方案**。

要决定的东西有：

```
1. 哪些铲位布置电铲；
2. 哪些铲位向哪些卸点运输；
3. 每条路线运输多少趟；
4. 每条路线安排几辆卡车；
5. 最终总运量、产量、品位是否满足要求。
```

这些“要决定的东西”以后就会变成**决策变量**。

------

## 第二步：找目标

数学建模里经常有一个目标：

```
最小什么？
最大什么？
最优什么？
```

这道题有两条生产原则，所以有两个模型。

### 模型一目标

题目第一条原则是：

```
总运量最小，同时出动最少卡车。
```

这里的主目标是：

```
总运量最小
```

附加目标是：

```
在总运量最小的基础上，卡车数尽量少
```

所以模型一不是单目标，而是**两阶段优化**。

------

### 模型二目标

题目第二条原则是：

```
利用现有车辆获得最大产量，岩石产量优先，产量相同时总运量最小。
```

这个目标有优先级：

```
第一优先级：岩石产量最大；
第二优先级：矿石产量最大；
第三优先级：总运量尽量小。
```

所以模型二是**分层优化模型**。

------

## 第三步：找约束

现实中不是你想怎么安排就怎么安排，因为有很多限制。

比如：

```
电铲只有 7 台；
卡车只有 20 辆；
每个铲位矿石和岩石储量有限；
每个卸点有最低产量要求；
矿石品位必须在 28.5% 到 30.5% 之间；
电铲装车有时间限制；
卸点卸车有时间限制；
卡车一个班次能跑的趟数有限；
同一路线车辆不能太多，否则会等待。
```

这些限制就是**约束条件**。

------

## 第四步：建立数学模型

数学模型一般由三部分组成：

```
决策变量 + 目标函数 + 约束条件
```

这道题就是典型的优化模型。

更准确地说，它是：

```
混合整数线性规划模型
```

这个名字听起来复杂，其实可以拆开理解。

### 1. 线性规划

意思是目标函数和约束条件都是一次式，没有平方、乘变量、指数这些复杂东西。

比如：

```
总运量 = 各路线运输趟数 × 距离 × 154 的和
```

这是线性的。

------

### 2. 整数规划

运输趟数不能是 3.6 趟，车辆数不能是 2.4 辆，所以必须是整数。

这就是整数规划。

------

### 3. 0-1 变量

有些事情只有“是”和“否”。

比如铲位 i 是否放电铲：

```
放电铲：1
不放电铲：0
```

这种变量叫 0-1 变量。

所以这道题里既有普通整数变量，又有 0-1 变量，因此叫**混合整数规划**。

------

# 三、重新理解这道题

这道题的本质是：

> 在 8 小时班次内，把有限的电铲和卡车分配到合适的铲位和运输路线，使矿石、岩石运输满足生产要求，并优化总运量或产量。

题目里有几类对象。

## 1. 铲位

铲位就是矿山中可以装矿石或岩石的地方。

题目有 10 个铲位：

```
铲位1，铲位2，……，铲位10
```

每个铲位都有：

```
矿石量
岩石量
矿石品位
```

例如某个铲位矿石多、品位高，另一个铲位岩石多、距离近，这些都会影响安排。

------

## 2. 卸点

卸点就是卡车把矿石或岩石运到哪里。

题目有 5 个卸点：

```
矿石漏
倒装场 I
倒装场 II
岩石漏
岩场
```

其中前三个是矿石卸点：

```
矿石漏、倒装场 I、倒装场 II
```

后两个是岩石卸点：

```
岩石漏、岩场
```

矿石不能运到岩石卸点，岩石也不能运到矿石卸点。

------

## 3. 电铲

电铲负责给卡车装车。

题目有 7 台电铲，但铲位有 10 个。

所以问题来了：

```
10 个铲位里，只能选最多 7 个铲位布置电铲。
```

这就是一个选择问题。

因此我们定义一个变量：

```
yi = 1，表示铲位 i 放电铲；
yi = 0，表示铲位 i 不放电铲。
```

------

## 4. 卡车

卡车负责运输。

题目有 20 辆卡车，每辆车载重 154 吨，也就是：

```
0.0154 万吨
```

卡车在某条路线上的工作过程是：

```
装车 → 从铲位开到卸点 → 卸车 → 返回铲位
```

所以一辆车完成一趟循环运输需要时间。

------

# 四、最核心的三个变量

这道题真正重要的变量只有三个。

## 变量一：xij

```
xij 表示铲位 i 到卸点 j 的运输趟数。
```

例如：

```
x_2,1 = 13
```

意思是：

```
铲位2 到 矿石漏 运输 13 趟。
```

这个变量决定了产量、运量、品位。

因为一趟车运 0.0154 万吨，所以：

```
运输量 = 0.0154 × 运输趟数
```

------

## 变量二：yi

```
yi 表示铲位 i 是否布置电铲。
```

如果：

```
yi = 1
```

说明铲位 i 有电铲，可以装车。

如果：

```
yi = 0
```

说明铲位 i 没有电铲，不能运输。

------

## 变量三：nij

```
nij 表示路线 i 到 j 上安排几辆卡车。
```

例如：

```
n_1,4 = 2
```

意思是：

```
铲位1 到 岩石漏 这条路线安排 2 辆车。
```

这个变量用来控制车辆数量和运输能力。

------

# 五、最重要的基础计算：循环时间

我们先理解一个非常关键的公式。

路线 i 到 j 的循环时间：

```
Tij = 5 + 3 + 2*dij/28*60
```

它表示：

> 一辆车在铲位 i 到卸点 j 这条路线完成一趟完整循环需要多少分钟。

拆开看：

```
5：装车时间，单位是分钟；
3：卸车时间，单位是分钟；
dij：铲位 i 到卸点 j 的单程距离，单位是 km；
2*dij：往返距离；
28：卡车速度，单位是 km/h；
乘 60：把小时换成分钟。
```

所以：

```
2*dij/28
```

得到的是小时。

因为前面 5 和 3 是分钟，所以必须乘 60，统一成分钟。

------

比如某条路线距离是 5.26 km：

```
Tij = 5 + 3 + 2*5.26/28*60
```

结果大约是：

```
30.54 分钟
```

意思是：

```
这辆车从装车开始，到卸货回来，大约需要 30.54 分钟。
```

------

# 六、一辆车最多能跑多少趟？

一个班次是 8 小时：

```
8 小时 = 480 分钟
```

如果一辆车跑某路线一趟需要 Tij 分钟，那么它最多能跑：

```
480 / Tij
```

趟。

因为不能跑半趟，所以取整数部分。

例如某路线循环时间是 30.54 分钟：

```
480 / 30.54 ≈ 15.7
```

所以最多跑：

```
15 趟
```

这就是：

```
Hij = 一辆车在路线 i 到 j 上最多运输趟数
```

------

# 七、总运量是什么？

题目说“总运量最小”，这里的总运量不是总产量，而是：

```
吨公里
```

吨公里的意思是：

```
运输重量 × 运输距离
```

一辆车一趟运 154 吨，路线距离是 dij km，所以一趟的吨公里是：

```
154*dij
```

如果这条路线跑 xij 趟，那么这条路线总运量是：

```
154*dij*xij
```

所有路线加起来，就是总运量 W：

```
W = 所有路线的 154*dij*xij 之和
```

模型一就是让 W 最小。

------

# 八、共同约束怎么理解？

两个模型虽然目标不同，但大部分约束是一样的。

## 1. 电铲数量约束

题目只有 7 台电铲，所以：

```
所有 yi 的和 <= 7
```

意思是：

```
最多只能选 7 个铲位放电铲。
```

------

## 2. 铲位装车能力约束

一台电铲装一车需要 5 分钟。

一个班次 480 分钟，所以一台电铲最多装：

```
480 / 5 = 96 趟
```

因此，对于每个铲位：

```
这个铲位运出去的总趟数 <= 96*yi
```

这条约束有两个作用。

如果 yi = 0：

```
总趟数 <= 0
```

说明没放电铲的铲位不能运输。

如果 yi = 1：

```
总趟数 <= 96
```

说明这个铲位最多装 96 车。

------

## 3. 矿石储量约束

每个铲位矿石量有限。

如果铲位 i 的矿石储量是 Oi，那么它运往矿石卸点的矿石量不能超过 Oi。

因为每趟是 0.0154 万吨，所以：

```
0.0154 × 铲位 i 运往矿石卸点的总趟数 <= Oi
```

------

## 4. 岩石储量约束

同理，岩石也不能超过储量。

如果铲位 i 的岩石储量是 Ri，那么：

```
0.0154 × 铲位 i 运往岩石卸点的总趟数 <= Ri
```

------

## 5. 卸点最低产量约束

每个卸点都有最低产量要求。

比如矿石漏至少要 1.2 万吨，岩石漏至少要 1.9 万吨。

所以对于每个卸点：

```
0.0154 × 所有铲位运到这个卸点的总趟数 >= 这个卸点的最低产量要求
```

这条约束保证任务完成。

------

## 6. 卸点卸车能力约束

卸一车要 3 分钟。

一个班次 480 分钟，所以一个卸点最多卸：

```
480 / 3 = 160 趟
```

因此：

```
运到某卸点的总趟数 <= 160
```

这条约束说明卸点不是无限服务的。

------

## 7. 品位约束

这是这道题比较难但很重要的一点。

矿石卸点要求平均品位在：

```
28.5% 到 30.5%
```

为什么需要这个约束？

因为不同铲位的矿石品位不同，有的 28%，有的 33%。

如果全运高品位矿，可能超过 30.5%；
 如果全运低品位矿，可能低于 28.5%。

所以必须进行搭配。

平均品位可以理解为：

```
混合平均品位 = 总含铁量 / 总矿石量
```

模型里用运输趟数加权。

为了让它变成线性约束，我们不用分式，而是写成两个不等式：

```
下限约束：所有 (gi - 0.285)*xij 的和 >= 0
上限约束：所有 (gi - 0.305)*xij 的和 <= 0
```

它们的作用就是保证平均品位在范围内。

------

## 8. 卡车运输能力约束

假设路线 i 到 j 安排了 nij 辆车。

每辆车最多跑 Hij 趟。

那么这条路线最多能完成：

```
Hij*nij
```

趟。

所以实际运输趟数不能超过它：

```
xij <= Hij*nij
```

这条约束保证：

```
路线上的运输任务不能超过车辆能力。
```

------

## 9. 卡车总数约束

一共只有 20 辆车，所以：

```
所有路线的 nij 之和 <= 20
```

------

## 10. 同一路线不等待约束

如果同一条路线车太多，可能出现车辆回来了，但前面车还没装完，后车就要等。

题目要求原则上不等待，所以要限制同一路线车辆数。

我们用：

```
nij <= Tij / 5 的整数部分
```

为什么除以 5？

因为电铲装一车需要 5 分钟。

如果一条路线循环时间很长，就可以安排更多车轮流跑；
 如果循环时间很短，车不能太多，否则回来太快会排队。

------

# 九、模型一完整理解

模型一的现实含义是：

> 在完成基本生产任务、品位合格、设备能力允许的情况下，让运输成本最低，并尽量少用车。

它分两阶段。

------

## 第一阶段：总运量最小

目标是：

```
min W
```

也就是：

```
让所有路线的 154*dij*xij 加起来最小
```

这个阶段会告诉我们：

```
怎样安排路线和运输趟数，总吨公里最小。
```

------

## 第二阶段：车辆数最少

第一阶段求出最小总运量 Wmin 后，第二阶段增加一个约束：

```
W <= Wmin
```

意思是：

```
总运量不能比第一阶段更差。
```

然后目标变成：

```
min N
```

其中：

```
N = 所有路线车辆数 nij 的和
```

这样就能得到：

```
在总运量最小的所有方案里，使用车辆数最少的方案。
```

这就是原则一的完整逻辑。

------

# 十、模型二完整理解

模型二的现实含义是：

> 现在有 20 辆车，尽量多生产，其中岩石产量优先。如果产量一样，再考虑运输成本低。

它也不是简单一个目标，而是三层目标。

------

## 第一层：岩石产量最大

岩石产量 PR 等于：

```
0.0154 × 所有运往岩石卸点的趟数之和
```

第一阶段目标：

```
max PR
```

先把岩石产量推到最大。

------

## 第二层：矿石产量最大

在岩石产量已经最大时，再让矿石产量最大。

矿石产量 PO 等于：

```
0.0154 × 所有运往矿石卸点的趟数之和
```

第二阶段目标：

```
max PO
```

同时要求：

```
PR 不能低于第一阶段得到的最大岩石产量
```

------

## 第三层：总运量最小

当岩石产量和矿石产量都已经达到最优后，再选一个总运量较小的方案。

第三阶段目标：

```
min W
```

所以模型二体现的是：

```
先保岩石，再保矿石，最后省距离。
```

------

# 十一、这道题为什么不是简单贪心？

很多初学者会想：

```
距离最近就运最近的，不就行了吗？
```

不行。

因为这道题同时有很多限制：

```
最近路线可能品位不合格；
最近铲位可能矿石储量不够；
最近卸点可能卸车能力已满；
某个铲位可能没有电铲；
某条路线可能车辆能力不够；
卡车总数只有 20 辆；
模型二还要优先岩石产量。
```

所以它不是简单选最近路线，而是一个整体优化问题。

这就是数学建模的价值。

------

# 十二、这道题的建模技巧

你作为建模小白，重点记住下面几个技巧。

## 技巧一：先问“我要决定什么”

决定什么，就设什么变量。

这道题决定：

```
是否放电铲 → yi
运输多少趟 → xij
派几辆车 → nij
```

------

## 技巧二：目标函数来自题目关键词

看到：

```
最小、最大、尽量少、尽量多、最优
```

就要想到目标函数。

这道题：

```
总运量最小 → min W
最大产量 → max P
车辆数最少 → min N
```

------

## 技巧三：所有现实限制都要变成约束

题目里的：

```
只有 7 台电铲
只有 20 辆车
每个卸点有产量要求
矿石品位有限制
每个卸点一次只能卸一辆车
每台电铲一次只能装一辆车
```

都要变成数学不等式。

------

## 技巧四：单位必须统一

这题里最容易错的是时间单位。

速度是：

```
km/h
```

装卸时间是：

```
分钟
```

所以运输时间要乘 60，把小时变成分钟。

另外载重是：

```
154 吨
```

产量单位是：

```
万吨
```

所以要换算：

```
154 吨 = 0.0154 万吨
```

这就是为什么模型里经常出现 0.0154。

------

## 技巧五：多目标问题不要乱加权

题目说：

```
总运量最小，同时车辆数最少
```

这不是直接：

```
min 总运量 + 车辆数
```

因为两个单位不同，一个是吨公里，一个是辆。

更规范的方法是分阶段：

```
先求主目标最优；
再在主目标最优的前提下优化次目标。
```

这就是分层优化。

------

# 十三、这道题最终模型的整体框架

你可以把整个模型理解成下面这个结构。

```
输入数据：
距离、矿石量、岩石量、品位、卸点需求、车辆数、电铲数、速度、装卸时间

预处理：
计算每条路线循环时间
计算每辆车每条路线最大趟数
计算每条路线单趟吨公里
```

对，你理解得很关键：**一般是先有数学模型，再根据模型写代码，最后通过代码调用求解器得到结果。**

但是要注意一句话：

> 数学模型负责“说明问题怎么被数学表达”，代码负责“把这个数学表达交给计算机求解”。

它们不是一回事。

------

# 1. 数学模型是什么？

数学模型是论文里的东西，是给人看的。

它回答的是：

```
这个问题要优化什么？
有哪些变量？
有哪些约束？
目标函数是什么？
为什么这样建模？
```

比如这道题中，数学模型会写：

```
xij 表示铲位 i 到卸点 j 的运输趟数；
yi 表示铲位 i 是否布置电铲；
nij 表示路线 i 到 j 上安排的卡车数量。
```

然后写目标函数：

```
模型一目标：总运量 W 最小
W = 所有路线的 154*dij*xij 之和
```

再写约束：

```
所有 yi 的和 <= 7
所有路线 nij 的和 <= 20
0.0154*运到每个卸点的趟数之和 >= 该卸点需求
xij <= Hij*nij
```

这些就是数学模型。

它的作用是：**把现实问题讲清楚、写成数学问题。**

------

# 2. 代码是什么？

代码是给计算机看的。

它回答的是：

```
怎么把模型输入电脑？
怎么调用求解器？
怎么让电脑算出 xij、yi、nij 的具体数值？
怎么导出结果？
```

比如模型中写：

```
xij 表示运输趟数
```

代码里就写成：

```
x = pulp.LpVariable.dicts("x", (I, J), lowBound=0, cat="Integer")
```

模型中写：

```
所有 yi 的和 <= 7
```

代码里就写成：

```
prob += pulp.lpSum(y[i] for i in I) <= 7
```

模型中写：

```
总运量最小
```

代码里就写成：

```
prob += pulp.lpSum(154 * d[i][j] * x[i][j] for i in I for j in J)
```

所以代码其实就是把论文里的数学模型翻译成 Python 语言。

# 3. 二者的关系

你可以这样理解：

```
现实问题 → 数学模型 → 代码实现 → 求解结果 → 论文分析
```

对应到这道题就是：

```

↓

↓

```

所以顺序通常是：

```

```

不是一上来就写代码。

# 4. 为什么不能只写代码？

因为数学建模比赛不是编程比赛。

评委主要看的是：

```
你是否理解了问题；
你是否合理建立了模型；
你的变量、目标函数、约束是否正确；
你的结果是否能解释现实问题。
```

代码只是用来帮你算结果的。

如果你只有代码，没有模型，评委会觉得：

```
你只是让电脑算了一堆数，但不知道你到底建了什么模型。
```

所以论文里必须有模型。

# 5. 为什么不能只写模型，不写代码？

因为这道题变量很多。

比如：

```

5 个卸点
20 辆卡车
```

运输路线最多有：

```
10 × 5 = 50 条路线
```

如果再考虑每辆车分配，就是：

```
10 × 5 × 20 = 1000 个可能的派车变量
```

靠手算根本不现实。

所以我们需要代码把模型交给求解器，让求解器自动寻找最优方案。

# 7. 求解器是什么？

求解器可以理解成一个“数学优化计算器”。

你把模型告诉它：

```
目标函数是什么；
变量是什么；
约束是什么；
哪些变量必须是整数；
哪些变量只能是 0 或 1。
```

然后它帮你找：

```
在所有满足约束的方案中，哪个方案目标最好。
```

我们代码里用的是 PuLP，PuLP 本身不是求解器，它更像一个建模接口。真正求解一般由 CBC 这类求解器完成。

可以理解为：

```
Python：负责组织数据和模型
PuLP：负责把模型写成求解器能懂的形式
CBC：负责真正求最优解
```

# 8. 论文里写模型，代码里实现模型，结果来自代码

这道题最终结果，比如：

```

使用 19 辆卡车
```

不是论文凭空写出来的，也不是手算出来的，而是代码根据模型求出来的。

流程是：

```
论文中建立模型
↓
代码中实现模型
↓
求解器计算最优解
↓
把结果返回给代码
↓
代码输出路线、车辆数、趟数、产量、品位
↓
论文中分析这些结果
```

# 9. 一个很重要的理解

数学模型和代码的区别可以用一句话说清：

> 数学模型是“为什么这样算”，代码是“具体怎么让电脑算”。

也可以这样记：

```
模型 = 思想、逻辑、公式
代码 = 实现、计算、输出
```

模型错了，代码写得再好也没用。
 模型对了，代码只是把它翻译出来。

# 11. 你作为论文手/模型手最应该掌握什么？

你不一定一开始就要完全掌握代码细节，但必须理解：

```
每个变量代表什么；
每个约束为什么要写；
目标函数为什么这样设计；
代码输出结果和模型之间是什么关系；
结果是否满足题目要求。
```

比如看到结果里：

```
铲位1 → 岩石漏，2辆车，81趟
```

你要能解释：

```
这是 xij 和 nij 的具体取值；
81 趟对应运输量 81*0.0154 = 1.2474 万吨；
2 辆车说明这条路线安排 nij=2；
这条路线满足车辆能力和不等待约束。
```

这样你就真的懂模型了。

------

总结一下：

**是的，要根据数学模型写代码，代码才能求出结果。**

但更准确地说：

> 数学模型决定代码应该写什么；代码只是模型的计算实现；求解结果来自代码对模型的求解。









有区别，但你先记住一句核心话：

**数学建模的重点不是“用什么软件”，而是“你能不能把模型变成可计算的形式”。**
 LINGO、MATLAB、Python 都只是实现工具。

# 五、数学模型和代码如何对应？

你可以记住这个对应表。

| 数学模型         | 代码                                   |
| ---------------- | -------------------------------------- |
| 集合 I、J        | `I = range(10)`, `J = range(5)`        |
| 参数 dij         | `d[i][j]`                              |
| 变量 xij         | `x[i][j] = LpVariable(...)`            |
| 0-1变量 yi       | `y[i] = LpVariable(..., cat="Binary")` |
| 目标函数 min W   | `prob += total_transport`              |
| 约束 sum yi <= 7 | `prob += lpSum(y[i] for i in I) <= 7`  |
| 求解模型         | `prob.solve()`                         |
| 输出结果         | `value(x[i][j])`                       |

所以你以后写代码时，不要乱写。
 你应该先看论文模型：

```
这个变量是什么？
这个约束是什么？
这个目标是什么？
```

然后一条一条翻译成代码。

# 六、初学者最容易犯的错误

## 错误一：还没建模型就开始写代码

很多小白一上来就问：

```
代码怎么写？
```

但其实应该先问：

```
我要设哪些变量？
我要优化什么？
有哪些约束？
```

模型没想清楚，代码一定会乱。

------

## 错误二：只会调库，不知道结果对不对

比如求解器输出一个结果，你不能直接信。

你要检查：

```
卡车有没有超过 20 辆？
电铲有没有超过 7 台？
卸点产量是否达标？
品位是否在范围内？
车辆能力是否满足？
总运量是否算对？
```

这就是结果验证。

------

## 错误三：单位不统一

数学建模里单位非常重要。

这道题就有：

```
吨 和 万吨
小时 和 分钟
单程距离 和 往返距离
```

如果单位错了，模型结果就错。

------

## 错误四：把多目标直接乱加

比如：

```
min 总运量 + 车辆数
```

这个不严谨，因为吨公里和辆不是一个单位。

更好的方法是：

```
先优化主目标；
再在主目标最优条件下优化次目标。
```

这就是我们说的分层优化。

# 十二、最后给你一个非常重要的学习方法

你以后学建模编程，不要单独背语法。
 要用这个方式学：

```
看到一个模型公式
↓
问自己：这个公式在代码里怎么写？
↓
看到一段代码
↓
问自己：它对应论文里的哪条变量、目标或约束？
```

比如：

```
prob += pulp.lpSum(y[i] for i in I) <= 7
```

你要马上反应出来：

```
这是电铲数量约束。
```

再比如：

```
prob += x[i][j] <= H[i][j] * n[i][j]
```

你要马上反应出来：

```
这是路线车辆运输能力约束。
```

这样你才是真的在学数学建模编程，而不是只是在复制代码。
